# 长江流域亚热带城市植物叶片滞尘数据集

**Plant Dust Retention — Yangtze Subtropical Urban Agglomeration**

植物叶片滞尘可行性验证与数据集构建。目标：为论文撰写提供"长江流域亚热带城市群植物叶片滞尘"的系统性数据基础。

- **GitHub**: https://github.com/wyq135/yangtze-plant-dust-retention
- **核心方法论**: **地理优先** —— 先锁定长江流域亚热带城市群，区域内所有植物物种的滞尘数据全量纳入，不再限制物种。物种趋同性由地理约束自然呈现。
- **方法论演变**: 物种优先（v1）→ 物种优先×跨功能区（v2）→ **地理优先×物种开放（v3，当前）**

## 物种纳入原则

**不限制物种。** 地理约束（长江流域亚热带）已足够让物种自然趋同。区域内任何植物的滞尘数据，只要满足数据纳入判据（单位叶面积定量数据、功能区可映射），一律收录。个别物种即使数据量少，只要可看出趋势就保留。

物种数不再设上限，预计最终可能达到 50-100+ 种。

## 协作方式

**主动使用并行子 agent 执行可并行任务。** 本项目大量工作具有天然并行性（文献搜索、数据提取、文档更新），单线程执行效率低下。遇到以下场景时，自行生成多个 subagent 并行推进，无需询问：

| 场景 | 并行策略 | 说明 |
|------|----------|------|
| **大规模文献搜索** | 每个搜索式一个 agent，5-8个并行 | 不同物种/城市/术语组合互不依赖 |
| **多论文同时提取** | 每篇论文一个 agent 独立提取 | 不同论文的数据互不依赖 |
| **文档多文件更新** | 每个文件一个 agent 并行读写 | CLAUDE.md/species_cards/data_sources 各自独立 |
| **数据验证与去重** | 一个 agent 审数据，一个 agent 改文档 | 发现→修正→写入流水线 |
| **多维度质量检查** | 每个维度一个 agent，3-5个并行 | 正确性/完整性/一致性/单位换算各司其职 |

**不适用并行的情况**：任务之间有先后依赖、单文件少量修改、简单的问答查询。

**经验来源**：2026-06-17 文献搜索与 Excel 导出任务中，并行生成 5 个搜索 agent + 1 个 Excel 生成 agent，完成速度远超串行。此后多轮任务中持续验证有效。

### Agent 止损规则（强制遵守）

Agent 必须有硬性退出条件，避免无限探索浪费时间和 token：

| 场景 | 止损条件 | 后续动作 |
|------|----------|----------|
| **PDF 下载** | 2 种方法失败 | 标记"需用户手动下载"，列出标题+链接 |
| **pdfplumber 数据提取** | txt 中表格区域无文本 | 立即标记"图片数据"，**禁止尝试 OCR** |
| **任何 agent** | 工具调用超过 30 次 | 汇报当前进展，停止继续探索 |
| **单论文提取** | 超过 5 分钟 | 汇报已提取/缺失情况，不继续深挖 |

**经验来源**：2026-06-18，下载 agent 用 58 次工具调用/14 分钟下 3 篇论文；刘庆 2009 提取 agent 用 116 次调用/43 分钟尝试 6 种 OCR 方案零成果。而用户用 DeepSeek 网页端图像识别 30 秒即完成。

### 图片表格处理流程（重要）

我通过 API 运行，**不支持图像识别**。用户可通过 DeepSeek 网页端使用原生多模态模型直接识别图片中的数据。

```
我发现图片数据 → 截图标注发用户 → 用户用DeepSeek网页端识别 → 文字回传 → 我入库
```

适用场景：PDF 中所有以图片形式嵌入的表格/柱状图数据。

### 论文下载分工

中文学术网站（知网/万方）对自动下载不友好，反爬严格。用户有大学内网直接访问权限。

**默认策略**：需要下载新论文时，我整理清单（标题+DOI/链接），用户手动下载后告知文件路径，我再提取数据。

### 零星数据提取（正文散布数值）

**核心教训**：论文正文中常提及精确数值——"X(种)最强(Y g/m²)"、"排序：A>B>C"——这些不在表格中但可直接利用。此前仅依赖表格提取，丢弃了大量正文零星数据。陈小平 2017 正文明确写"杜鹃最强（179.41 µg·cm⁻²）"即 1.79 g/m²，但此前被忽略。

**工具**：`extract_scattered_data.py` — 自动扫描所有 TXT 文件，正则匹配「物种名 + 数字 + 单位」组合，过滤已入库数据，输出候选清单。

**工作流程**：
```
每次新提取 PDF 文本后 → 运行 extract_scattered_data.py
                   → 人工甄别输出（排除冠层面积 g·m⁻²·crown⁻¹、单叶 g·leaf⁻¹）
                   → 确认的数值入库
```

**常见陷阱**：
| 陷阱 | 说明 | 处理 |
|------|------|------|
| 冠层面积 | `g·m⁻²·crown⁻¹` 是单位树冠投影面积，非单位叶面积 | 排除 |
| 单叶重量 | `g·leaf⁻¹` 是单叶总滞尘量 | 排除（除非同时有叶面积） |
| PM 粒径分数 | PM10/PM2.5 值常被误标为 TSP | 核对原文上下文 |

**图片识别分工**：
- **文本表格** → DeepSeek 网页端多模态识别 → 用户回传 → 我入库
- **柱状图/饼图** → 无法精确读数 → 放弃，仅依赖正文零星数值
- **经验来源**：2026-06-18，高丹丹文本表格识别成功（确认已有数据），陈小平/张冰洁柱状图识别失败

## 已知局限（写论文时需注意）

1. 地被层30条记录，仍远少于乔木(145)和灌木(88)
2. 不同研究采样间隔（雨后天数）差异大（1-25天），未完全标准化
3. 部分环境背景数据（ambient PM、温湿度）为论文原文缺失后的估计值
4. ~~广州/深圳~~ 已排除（非长江流域）
5. 物种覆盖不均：二球悬铃木(19)仍远少于香樟(40)/桂花(42)；沿阶草(1)/洒金桃叶珊瑚(6)仍偏少
6. **功能区名称不一致**：`交通区`/`交通干道`、`清洁区`/`公园清洁区`、`街道绿地`/`新兴区`混用，论文写作前需统一
7. **PM10/PM2.5 数据不完整**：多数早期研究仅测TSP。PM10/PM2.5数据集中于王琴(2020)、张冰洁(2024)、Zhang BJ(2025)、贺丹(2025)、Chen Yazhen(2023)五篇
8. **柱状图读数可能偏差**：数据集中部分TSP值从论文柱状图人工读数获取，发现并修正了1处错误（红花檵木 交通区 4.96→2.83），不排除其他类似偏差
9. **测量方法不统一**：Chen Yazhen 2023 使用水溶消解法（数值偏低5-10倍），叶志群2023 使用气溶胶再发生器法（PM2.5值比超声清洗法低约10-50倍），不可与其他论文直接合并分析
10. **麦冬/沿阶草混用**：中文文献常混用 Ophiopogon japonicus/bodinieri，需逐条核实
11. **柱状图数据无法利用**：多篇论文（Changsha overpasses、Sheng 2024、晏增 2021）数据仅在无标注柱状图中，无法精确读数
12. **μg/cm²→g/m²换算错误已修正**（2026-06-17）：Zhang BJ 2025(16条)和张冰洁2024(22条)的PM10/PM2.5值原用÷10错误换算（应为÷100），已÷10修正。涉及64个数值，TSP未受影响

## 文献元数据抓取

### 问题诊断

Claude Code 内置 WebFetch 无法访问中文学术网站，**根因是 Anthropic 服务端对 `.cn` 域名有白名单过滤**，与本地网络、VPN 无关。用户无法配置绕过。

### 三条技术路线对比

| 路线 | 工具 | 中文学术网站 | 国际期刊 | 结论 |
|------|------|:--:|:--:|------|
| 1 | Claude WebFetch | ❌ 全部被拦 | ⚠️ 部分可用 | 服务端域名白名单，不可控 |
| 2 | Firecrawl API | ❌ JS劫持 | ⚠️ 付费墙 | 浏览器渲染被挂马网站劫持 |
| 3 | **curl + Python** | ✅ 可用 | ⚠️ 付费墙 | **首选方案** |

**路线2 失败原因**：`ecologica.cn` 等中文学术网站被大面积挂马，检测到浏览器 User-Agent 后注入恶意 JS 重定向到色情广告域名。Firecrawl 使用真实浏览器渲染，JS 劫持生效。curl 不执行 JS，天然免疫。

### 中文 DOI 解析路径

```
doi.org → chndoi.org（中文 DOI 注册中心）→ link.cnki.net（知网，付费墙）
                ↓
        "多重解析地址选择页面"
        含：题名、作者、DOI
        不含：摘要、期刊名 ← 需要知网认证
```

- **chndoi.org** 是中文 DOI 注册中心，介于 doi.org 和知网之间
- 提供论文标题和完整作者列表（可修正数据集中"X等"的简写）
- 期刊名被 CNKI 出版商信息覆盖，需从其他来源补充
- 完整摘要和数值数据需通过学校图书馆/知网获取

### scrape_paper.py 使用方法

```bash
cd plant_dust_analysis

# 基本用法：通过 DOI 抓取
python scrape_paper.py "10.5846/stxb201808241808"

# 保存为 JSON
python scrape_paper.py "10.13275/j.cnki.lykxyj.2021.04.010" --json -o meta.json

# 也支持直接 URL
python scrape_paper.py "https://www.ecologica.cn/stxb/article/abstract/201808241808"
```

**技术流程**：
```
DOI/URL → curl -sL（跟随重定向，伪装 Chrome UA）
       → 自动编码检测（UTF-8 → GB18030 → GBK → GB2312）
       → 正则清洗 HTML（去标签、解实体）
       → 结构化 JSON（标题/作者/摘要/期刊/DOI/基金/卷期页）
```

**关键设计决策**：
- 编码检测按优先级依次尝试，以页面中是否出现「摘要/标题/DOI」等关键词判断解码成功
- chndoi.org 的"多重解析地址选择页面"有专用解析逻辑
- 作者提取优先用"引用本文"格式（中文姓名连续出现），而非表单标签
- 期刊名通过引用格式 `期刊名, 年份, 卷(期): 页` 正则提取

## 文献搜索方法论（2026-06-16 经验总结）

### 已识别的严重错误

**错误1：系统性忽略硕博论文**

v2 数据集的早期 18 篇参考文献中仅含 1 篇硕士论文（肖慧玲2013），但硕博论文是滞尘数据密度最高的文献类型——单篇可贡献 5-20 条记录，有完整的实验设计章节和原始数据表格（期刊论文常因篇幅限制压缩数据）。华中农业大学 2024 年一篇硕士论文（135 页，导师朱春阳）覆盖红花檵木/麦冬/海桐/桂花/悬铃木 5 个目标物种，此前完全未被检索到。

**根因**：WebSearch 搜索式未区分文献类型，默认搜到的排在前面的都是期刊论文。未专门搜索 `site:cnki.net 硕士论文` 或访问万方学位论文库。

**错误2：把"知网付费墙"固化为不可逾越的约束**

项目 CLAUDE.md 中写了"知网付费墙→无法获取全文数据"，实际上是用户电脑接入大学内网、拥有完整下载权限。这个假设从未被验证就被当作硬约束。

**根因**：混淆了"当前工具有限"（WebFetch 无法访问 .cn）和"实际网络环境"（用户电脑有学校内网）。

**错误3：搜索维度单薄**

仅按"物种名 + 滞尘术语"搜索，遗漏了：
- **机构搜索**：华中农业大学、南京林业大学、中南林业科技大学、浙江大学、安徽农业大学、江西农业大学、四川农业大学、西南大学、湖南农业大学等长江流域机构
- **导师跟踪**：朱春阳团队在武汉有系统性的滞尘研究，包括多篇 SCI 和硕士论文
- **引用追索**：未从已知论文（肖慧玲2013、王琴2020）追踪其参考文献和施引文献
- **功能区位词**：很多论文标题里写"道路""公园""工业区"而不写"功能区"

**错误4：数据库覆盖不完整**

仅覆盖了 Web of Science 和知网部分内容，万方数据（wanfangdata.com.cn）有大量独家收录论文，未充分检索。

### 正确的文献搜索方法论（强制遵守）

1. **五维度搜索矩阵**（每个城市执行）：
   - 城市名 + 滞尘术语（基础搜索）
   - 机构名 + 滞尘术语（机构搜索）
   - 已知核心作者/导师名 + 滞尘（团队跟踪）
   - "城市名" + "植物/绿化" + "滞尘/颗粒物" + "硕士/博士"（学位论文专项）
   - 期刊名 + 滞尘 + 城市（高频期刊定向）

2. **数据库优先级**：
   - P0: 知网 CNKI（含硕博论文库 CDMD）+ 万方数据（含独家学位论文）
   - P1: Web of Science（国际期刊）
   - P2: 维普 CQVIP + 百度学术 xueshu.baidu.com
   - P3: Google Scholar

3. **引用网络追踪**（每篇纳入论文必做）：
   - 检查其参考文献列表 → 发现前置研究
   - 检查施引文献（谁引用了它）→ 发现后续研究
   - 同导师/同机构的论文 → 发现团队系列研究

4. **文献类型权重**（数据密度排序）：
   - 硕士论文 > 博士论文 > 中文核心期刊 > 普通期刊 > 会议论文

5. **功能区位词扩展**：
   论文中可能不写"功能区"，而是直接写具体位置。搜索时需涵盖：
   `道路|交通|街道|工业区|工厂|公园|校园|绿地|居住区|小区|清洁区|郊区`

### 其他已检查的潜在错误

- **单位换算错误**：未发现。mg/cm² → g/m² 的 ×0.1 换算已在所有记录中正确应用。
- **功能区归类一致性**：存在需重新审查的可能性——部分记录的 "城市混合" 在原始论文中可能是明确的道路/公园数据，应追溯原文逐条核实。
- **物种拉丁名准确性**：未发现错误，但麦冬/沿阶草两个近缘种在中文文献中常混用，需逐条标注实际鉴定种。
