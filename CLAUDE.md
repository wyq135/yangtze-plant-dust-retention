## 🛑 工程纪律与红线（2026-06-23 制定）

### 1. 数据序列化：禁止裸字符串CSV
- **绝对禁止**：使用 `f"{a},{b}\n"` 或 `','.join()` 手动拼接CSV行。
- **强制规范**：所有CSV读写必须使用 `csv.reader`/`csv.writer`，写入时指定 `quoting=csv.QUOTE_MINIMAL`。
- **验证**：输出后立即 `csv.reader` 回读验证列数一致。
- **原因**：`reference` 字段含多作者逗号（如"屈新运, 张天翼, 高天娥, 等."），裸字符串拼接必导致列偏移。v4两批841条全部因此返工。

### 2. 子进程调用：显式编码
- **绝对禁止**：Windows上 `subprocess.run(capture_output=True, text=True)` 不指定编码。
- **强制规范**：必须 `encoding='utf-8', errors='replace'`。或优先用 Python 原生库（pdfplumber）替代子进程。
- **原因**：Windows默认GBK，Python工具输出UTF-8，32篇PDF提取全灭。

### 3. 熔断机制：3次失败即停
- **规则**：同一方案连续3次不同根因失败 → 立即停止 → 标记为"需人工介入" → 生成待办清单。
- **禁止**：反复调试同类方法、换参数重试同一接口、同问题超过1小时无进展仍不切换。
- **原因**：CDP知网下载耗费200K+ token，4种方案均失败，根因是知网多层反爬而非技术bug。

### 4. CDP 使用边界（修订）
- **允许**：搜索文献、提取元数据、读取页面内容（只读操作）。
- **禁止**：通过CDP触发下载、Python requests代Cookie下载知网PDF。
- **替代**：生成下载清单 → 用户手动下载 → pdfplumber提取。

### 5. 文件分类：基于生命周期，禁止临时起意
- **绝对禁止**：看到一种新情况就新建一个目录（`_不清晰/`、`_纯图表跳过/`等）。
- **强制规范**：所有PDF只进入6状态互斥分类（`_待处理/` → `_需VLM/` → `_需重下/` → `_待人工/` → `_已入库/` → `_已排除/`）。
- **保守原则**：不确定有没有数据 → 放 `_待人工/`，绝不放 `_已排除/`。
- **原因**：临时建目录导致文件散落多处、新会话找不到文件、同一论文出现在多个目录。

### 6. 数据统计：不手写数字，从数据集生成
- **强制规范**：CLAUDE.md 中只保留概括性描述，不写具体统计数字。
- **验证**：需要统计时运行 `python .claude/_gen_stats_report.py`（从 dataset.csv 实时生成）。
- **原因**：手动维护的数字必然过时（今晚发现method_level/needs_manual_review等数字全是几周前的）。

## 已知局限（写论文时需注意）

1. **采样周期不统一**: 不同论文采样间隔差异大（日均值 vs 3天 vs 5天 vs 7天 vs 12天 vs 15天累积），均值7.4d（范围3-28d），合并分析需按累积天数做标准化或亚组分析
2. **测量方法不统一**: Chen Yazhen 2023 水溶消解法（数值偏低5-10倍），吴翠蓉2020/叶志群2023 气溶胶再发生器法（TSP/PM10/PM2.5均低10-50倍），不可与其他论文直接合并
3. **PM10/PM2.5数据不完整**: PM10仅234条(10.6%)，PM2.5仅243条(11.0%)，多数早期研究仅测TSP。PM分级数据集中于少数近年论文
4. ~~**功能区名称不一致**~~: ✅ 已标准化为9类（交通干道/道路绿地/公园清洁区/城市混合/文教区/工业区/居住区/室内模拟/跨功能区均值+未区分），2026-06-23 QC完成
5. **地被层/藤本数据偏少**: 地被31条(1.4%)+藤本8条，远少于乔木(50.7%)和灌木(39.0%)
6. ~~**μg/cm²→g/m²换算**~~: ✅ 2026-06-17已完成全部修正（÷100非÷10），涉及Zhang BJ 2025和张冰洁2024共64个PM数值
7. **中小城市数据空白（v4更新）**: v3确认无锡/常州/宁波/绍兴/南通/马鞍山/铜陵/黄石/宜昌/九江/自贡等11城无植物叶片滞尘数据。v4新增阿克苏(121条)/南宁/厦门/昆明/湛江/福州/唐山/石家庄等，城市覆盖增至39个，但长江中下游中小城市数据仍稀缺。镇江259条均为张翼飞2013 mg/leaf估算转换(C级)
8. **雨后5d均值异常偏高**: 雨后5d的TSP均值(5.32 g/m²)反而高于雨后7d(2.29 g/m²)，主要由谢英赞（北碚52条）贡献，反映论文间方法学混杂（非单纯累积时间效应），合并分析时需按论文做亚组分析
9. **张灵艺2015极高值**: 3条TSP>50 g/m²（雀舌黄杨61.5/麦冬86.3/细叶结缕草162.8），均为长累积期(15-25d)+地被植物，统计时应做敏感性分析（含/不含极端值）
9b. **张莉2007冬季极端值**: 冬季TSP达10.8-736.8 g/m²（红花檵木736.8/金叶女贞481.0/小叶黄杨470.5），虽论文标注"雨后第4天"但南京12月实际降雨极少，累积期可能远超4天。统计时应标注此不确定性。
10. **构树冬季零值**: 3条构树TSP=0（成都/自贡/遂宁），原因为2月冬季落叶期叶片掉光（江欣燕2016原文："2月叶片基本掉光，故滞尘量2月为0g/m²"），数据有效但论文中需标注物候原因
11. **朱天燕2018全年最大值**: 26条TSP为四季取max（非单季均值），灌木最高1398 g/m²。代表物种最大滞尘潜力，不可与其他论文单季均值直接比较，分析时需单独标注。
12. **张翼飞2013单位估算**: 259条原始单位mg/leaf，通过属级平均单叶面积(c㎡)估算转换为g/㎡（标B级, needs_manual_review=True），转换不确定性因属而异（叶面积跨6-300 cm²），与直接测量g/㎡的记录合并需谨慎。统计分析建议：亚组分析或标注为估算数据
13. **148条季节未标注**: 18篇论文未报告具体采样季节，标注为"未标注"。来源论文多为早期（2001-2010）或地方期刊，方法学描述较简略。
14. **Ficus小叶榕分类学歧义**: 中文名"小叶榕"在不同论文中对应不同拉丁名（Ficus microcarpa/concinna/benjamina），FRPS接受名为F. concinna（雅榕），F. microcarpa为榕树（不同物种），F. benjamina为垂叶榕。2026-06-23已将13条统一为F. concinna，notes留痕。但原始论文作者的野外鉴定本身可能存在歧义，统计分析时需标注此不确定性。

### 论文下载工作流（修订）

**Phase 1: 搜索与去重**（CDP只读）
1. Edge登录知网 → DevTools MCP接管 → 搜索关键词
2. 抓取结果列表 → 去重 → 输出下载清单

**Phase 2: PDF获取**（用户手动）
- Claude生成下载清单 → 用户在知网手动下载 → 放入 `_待提取/`
- ⚠️ CDP下载已证伪（知网CORS+Cookie+token多层反爬），不重试

**Phase 3: 智能提取**（全自动）
1. pdfplumber 提取文本层（**不用subprocess，直接调API**）
2. 并行 Agent 提取数据（CSV必须用csv.writer）
3. Python后处理：单位换算、物种标准化、method_level标注

**Phase 4: 人工QC**（专注异常）
1. 筛选 `method_level='C'` 或 `needs_manual_review=True`
2. 仅人工核对异常记录

### v4 数据入库强制字段

提取 Agent 输出必须包含以下 v4 专属字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| `method_level` | A/B/C 方法学等级 | A |
| `original_unit` | 论文原始单位 | mg/cm² |
| `conversion_log` | 换算过程记录 | "÷100 → g/m²" |
| `accepted_name` | 物种接受名（标准化后） | Osmanthus fragrans |
| `needs_manual_review` | 是否需人工复核 | false |
| `climate_zone` | 气候区（v4分层） | 北方/南方/西北 |

### 调用优先级（修订）

1. **CNKI三库 → Chrome DevTools MCP**（不复用登录态就是浪费）
2. **能用本地就不用远程**: pdf_extractor/VLM > WebFetch/WebSearch
3. **批量任务必须并行 Agent**: 3+ 论文提取同时启动
4. **VLM 只跑一次**: 失败即标记 needs_manual_review=True，不重复 OCR
5. **复杂查询走 SQLite MCP**: 去重、跨维度聚合；简单过滤走 pandas
6. **英文论文 → paper-search Skill**: 22源，自带下载和文本提取

### 工具局限性认知（修订）

| 工具 | 已知局限 | 应对 |
|------|------|------|
| Chrome DevTools | 需用户手动登录；部分论文仅CAJ/需滑块 | 异常自动标记到 error log，用户批量手动 |
| VLM (3B) | 中文表格 OCR 精度不足 | 仅做初筛，method_level默认C，人工确认 |
| Playwright MCP | 无登录态，知网会拦截 | 仅用于百度学术/国外网站 |
| paper-search | 无中文数据库 | 仅用于英文论文 |
| Crawl4AI | JS渲染页面效果差 | 仅用于静态页面 |
| WebFetch | 被半数网站拦截 | 降级到 Playwright MCP |
